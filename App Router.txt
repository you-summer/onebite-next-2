이제는 페이지라우터를 활용하지 않기 때문에 pages폴더는 존재하지 않음!

대신에 app router를 활용하기 때문에 app이라는 폴더가 있는걸 볼 수있음
app폴더밑에 globals.css
layout.tsx 라는 레이아웃설정하는 파일
page.tsx 파일이있음

앱라우터 버전의 넥스트앱에서는 페이지라는 이름을 갖고있는 파일이 페이지역할을 하는 파일이 됨
layout.tsx는 페이지의 레이아웃 역할을 하는 파일이 될거임
자세히 살펴볼 필요는 없고 그냥 그런식으로 설정이 되는구나 정도로 살펴보면 될거임

페이지 라우팅은 pages폴더 기반으로 라우팅을 설정해줌
앱 라우팅은 app 폴더를 기반으로 라우팅을 설정하게됨

page router 버전에서는 pages폴더 아래에 index.tsx 파일은 index 페이지가 되고 
search.tsx 파일은 search 페이지가 되는것 ㅓ럼
pages 폴더 아래에 생성해둔 이 파일들의 이름이 곧 경로가 되었었던 반면 app router 버전에서는 더 이상 그렇진 않음

app router에서는 대신에 무조건 page라는 이름을 갖는 파일만 이렇게 페이지 파일로서 취급됨
page.tsx

그래서 /search 에 대응하는 파일을 만들려면
app폴더 아래/search 폴더를 만든 다음에 그 폴더 아래에 page.tsx를 만들어 주면됨

~/book
app/book/page.tsx 파일을 만들어 주면 됨

book/1 , book/2 이런 동적경로를 대응하는 페이지를 만들 때에도
app/book/[id]/page.tsx 이렇게 만들어주면됨

app router 버전의 넥스트 앱에서도 앱 폴더 밑에 구조를 기반으로 페이지라우팅이 자동 설정된다는 점은 페이지 라우터 버전과 동일하지만 앱라우터에서는 페이지라는 이름을 갖는 파일만 페이지 파일로서 설정이 됨!!

이런 쿼리스트링은 페이지 컴포넌트 내부에서 사용하는 방법!
쿼리스트링은 경로의 끝에 물음표와 함께 명시가 되기 때문에 라우팅에는 전혀 영향을 주지 않음
우리 프로젝트에서도 별도로 쿼리스트링을 위해 폴더 구조를 변경해야된다거나 그러진 않음

쿼리스트링을 꺼내 쓰려면 앱라우터에서는 페이지 컴포넌트에게 자동으로 전달되는 props를 통해서 props에서 꺼내서 사용하면됨
앱라우터 버전의 넥스트 앱에서는 쿼리스트링이나 url파라미터와 같은 경로상에 포함되는 값들은
페이지 컴포넌트에게 props로서 전달이 됨
그렇기 때문에 우리가 코드 상에서 Page(props)를 받아와서  콘솔로 찍어보면

searchParams 객체로 부터 쿼리 스트링의 값을 꺼내서 활용하려면
type정의를 해줘야함

페이지 컴포넌트의 props로 searchParams 라는 객체가 들어올텐데
searchParams의 타입은 Promise 객체 형태로 그 안에 q는 string라는 결과값을 포함하고 있다
라고 타입을 정의해주면됨

export default async function Page({
  searchParams,
}: {
  searchParams: Promise<{ q: string }>;
}) {
  //   console.log(props);
  const { q } = await searchParams;
  return <>Search 페이지!!</>;
}

리액트의 서버컴포넌트라서 function Page()에 async를 붙일수있는거임
서버 컴포넌트의 경우 서버 측에서 사전렌더링을 위해서만 딱 한번 실행이됨
그래서 비동기적으로 실행이 되어도 전혀 문제가 발생하지 않음


앱라우터의 레이아웃 설정하기
app폴더 안에 page.tsx 파일을 배치해주면 해당파일은 인덱스 페이지로써 작동함
/app/search/page.tsx 해당파일은 경로에 맞게 searchㅠㅔ이지로써 작동하게 된다!

앱라우터에서는 page라는 이름의 파일이 페이지 역할을하는 파일로 자동 설정이 됨

레이아웃도 마찬가지!
/app/search/page.tsx
                 layout.tsx
이렇게 되면 search페이지의 레이아웃으로 자동 적용됨
그래서 search페이지는 레이아웃 컴포넌트가 먼저 렌더링 되고 나서 
그 안에 칠드런으로 page.tsx 페이지 컴포넌트가 렌더링 됨

그래서 레이아웃이라는 이름의 파일을 만들게 되면 해당 파일에 작성한 컴포넌트가 해당하는 경로의 layout으로써 자동 설정이됨
이렇게 설정된 레이아웃은 현재 search페이지 뿐만 아니라 그냥 /search 경로로 시작하는 모든 페이지의 레이아웃으로 적용됨
만약에 search페이지 아래에 setting이라는 폴더를 추가하고
app/search/setting/page.tsx 
이 페이지에도 역시 앞서 만들어둔 레이아웃이 함께 적용됨 /search로 시작하는 경로이므로 레이아웃 자동 적용

/search/layout.tsx
